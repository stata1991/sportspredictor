## 2. Backend changes

### New route: `GET /api/football/head-to-head`

**File:** `backend/football/routes.py:273-322`

```python
@router.get("/head-to-head")
async def get_head_to_head(
    response: Response,
    team1: int = Query(...),
    team2: int = Query(...),
    last: int = Query(5, ge=1, le=10),
    client: APIFootballClient = Depends(get_football_client),
) -> dict:
```

Parameters:
- `team1` (required): API-Football team ID for the home side
- `team2` (required): API-Football team ID for the away side
- `last` (optional, default=5, max=10): number of recent meetings

Response shape:
```json
{
  "team1_id": 6,
  "team2_id": 25,
  "count": 2,
  "summary": {"wins": 1, "draws": 0, "losses": 1},
  "fixtures": [...]
}
```

Summary is computed from team1's perspective — a "win" means team1 won regardless of home/away placement in that fixture.

### Cache TTLs

- **Server (data_provider):** 21,600s (6 hours) — `ENDPOINT_TTLS["headtohead"]`
- **CDN (Cache-Control):** 3,600s (1 hour) — `_CC_H2H = 3600`

### Team IDs added to prediction response

All three return paths in `predict_pre_match` now include `home_team_id` and `away_team_id` fields:
- Completed fixture response (line ~412)
- Cached prediction response (line ~455)
- Fresh prediction response (line ~556)
